# Форматы Экспорта

Поддерживается экспорты данных в JSON, CSV и Markdown.

## JSON (.json)

Самый полный формат, содержащий всю доступную информацию. Идеален для машинной обработки.

### Пример структуры (Course)

```json
{
  "course_id": 12345,
  "title": "Название Курса",
  "teachers": ["Преподаватель 1", "Преподаватель 2"],
  "tasks": [
    {
      "task_id": 67890,
      "title": "Задача 1",
      "description": "<p>Текст условия...</p>",
      "deadline": "2024-12-31T23:59:00",
      "max_score": 10.0,
      "score": 8.5,
      "status": "Зачтено",
      "section": "Раздел 1",
      "edit_url": "/task/edit/67890",
      "submit_url": "/submit/67890"
    }
  ]
}
```

## CSV (.csv)

Табличный формат, который можно открыть в Excel или Google Sheets.

### Колонки для задач (Student View)

1. `#` - Номер по порядку.
2. `Title` - Название задачи.
3. `Score` - Оценка студента.
4. `Status` - Статус (например, "На проверке").
5. `Deadline` - Дедлайн.

### Колонки для очереди (Queue)

1. `Student` - Имя студента.
2. `Task` - Название задачи.
3. `Status` - Статус проверки.
4. `Reviewer` - Проверяющий преподаватель.
5. `Updated` - Дата последнего обновления.
6. `Grade` - Оценка.

### Колонки для ведомости (Gradebook)

Таблица формируется динамически:

- `Group` - Группа студента.
- `Student` - Имя.
- `Task 1`, `Task 2`, ... - Оценки за каждую задачу.
- `Total` - Сумма баллов.

Вы можете выбирать, какие колонки включать в CSV, используя аргументы `--include-columns`.

## Markdown (.md)

Удобен для просмотра на GitHub или в IDE.

- Содержит заголовки с названием курса.
- Сводную таблицу (Tasks или Queue).
- В случае экспорта задач включает полные тексты условий (если они были загружены).

## Files Only (Только файлы)

Специальный режим экспорта для очереди (`queue`).

- Не создает единый файл-отчет.
- Создает структуру папок: `Output_Dir / Student_Name / Task_Name`.
- Скачивает внутрь все файлы, приложенные студентом в комментариях к решению.
- Ссылки на Google Colab скачиваются в формате `.ipynb`.
